# 寓管家租房经营小程序 MVP 用户手册

## 1. 产品定位

寓管家是一个面向二手房东、包租运营团队的租房经营小程序 MVP。当前版本主要用于演示“房源管理、经营统计、待办处理、房源录入”等核心流程，风格参考自如类租房小程序。

当前小程序首页是“应用中心”，包含三个入口：

- AI P 图 / 生成图片
- AI 宠物面相
- 寓管家租房经营 MVP

本手册只说明“寓管家租房经营 MVP”的使用方式。

## 2. 启动方式

### 2.1 启动租房后端

在项目根目录下执行：

```bash
cd server
npm install
npm run dev
```

租房后端默认地址：

```text
http://localhost:3010
```

说明：如果未启动后端，小程序端也内置了演示数据，可以进行基础页面演示；但新增房源、处理待办等数据持久化能力建议配合后端使用。

### 2.2 启动微信小程序

1. 打开微信开发者工具。
2. 选择项目目录：`D:\ai-pet-face-miniapp\miniprogram`。
3. 开启“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”。
4. 点击“编译”。
5. 首页进入“应用中心”，点击“寓管家租房经营 MVP”。

## 3. 角色说明

MVP 当前支持以下演示角色：

| 角色 | 主要用途 |
| --- | --- |
| 访客 | 只看公开房源，联系人、投入、收益等经营字段脱敏；不能看统计和待办 |
| 管理员 | 查看全部房源、全部统计、全部待办，可新增和维护房源 |
| 投资人 | 查看全部房源和经营统计，联系人脱敏；不能处理待办 |
| 收房员 | 只看收房相关房源和收房待办，可新增房源；租客信息脱敏 |
| 出租员 | 只看可出租/已出租房源和出房、合同、租金待办；业主与投入信息脱敏 |

当前版本主要通过“我的”页面切换身份，用于演示不同业务视角。权限隔离是 MVP 级别，后续正式版可继续强化。

## 4. 页面功能

### 4.1 应用中心

路径：`pages/index/index`

用于选择进入哪个业务模块：

- 点击“AI P 图 / 生成图片”进入原 AI P 图功能。
- 点击“AI 宠物面相”进入原宠物报告功能。
- 点击“寓管家租房经营 MVP”进入租房经营模块。

### 4.2 租房主页

路径：`pages/home/home`

主要用于浏览房源。

功能：

- 查看推荐房源列表。
- 按房源状态筛选：
  - 全部
  - 已出租
  - 空置
  - 待审核
- 按小区、房号、业主搜索。
- 点击房源卡片进入房源详情。
- 点击右上角“+”新增房源。
- 点击快捷入口进入：
  - 统计
  - 待办
  - 我的

房源卡片展示内容：

- 房源名称
- 房源状态
- 地址
- 面积、户型、朝向
- 月租金
- 收益率

### 4.3 房源详情

路径：`pages/room/room`

用于查看单套房源的完整信息。

展示内容：

- 房源图片
- 房源名称与地址
- 出租状态
- 月租
- 总投入
- 收益率
- 户型、面积、楼层、朝向
- 收房员、出租员
- 房源配置
- 业主联系方式
- 租客联系方式
- 房源备注

可执行操作：

- 点击“电话”联系业主。
- 点击“电话”联系租客。
- 点击“查看相关待办”进入经营待办页面。

### 4.4 经营统计

路径：`pages/dashboard/dashboard`

用于展示运营数据概览。

当前指标：

- 总房源数量
- 在租数量
- 空置数量
- 待办数量
- 当月收入
- 当月支出
- 当月利润
- 出租结构
- 月度收支柱状展示

该页面适合管理员、投资人在演示时查看整体经营情况。

### 4.5 经营待办

路径：`pages/manage/manage`

用于处理日常经营事项。

待办类型：

- 全部
- 收房
- 出房
- 合同
- 租金

待办卡片展示：

- 待办类型
- 处理状态
- 待办标题
- 客户名称
- 截止时间
- 金额

可执行操作：

- 点击“处理”将待办标记为已处理。
- 点击待办内容进入对应房源详情。
- 点击“新增房源”录入新房源。
- 点击“统计”进入统计页面。
- 点击“我的”进入个人中心。

### 4.6 新增房源

路径：`pages/room-form/room-form`

用于录入一套新的房源。

必填项：

- 小区名称
- 房号
- 租金

可填项：

- 房源地址
- 户型
- 面积
- 业主姓名
- 业主电话
- 备注

提交后：

- 房源状态默认为“待审核”。
- 后端会生成一条“新房源待审核”待办。
- 页面会跳转到新房源详情页。

### 4.7 我的

路径：`pages/me/me`

用于查看当前演示身份和进入个人相关功能。

功能：

- 查看当前角色。
- 进入“我的工单”。
- 进入“房源管理”。
- 进入“收支统计”。
- 查看帮助与客服。
- 切换身份。

客服演示电话：

```text
0755-88888888
```

## 5. 推荐演示流程

### 流程一：管理员经营总览

1. 从应用中心进入“寓管家租房经营 MVP”。
2. 在租房主页浏览房源列表。
3. 点击“统计”查看总房源、在租、空置、待办和月度收支。
4. 返回后点击“待办”。
5. 查看收房、出房、合同、租金等待办。
6. 点击“处理”完成一条待办。

### 流程二：录入新房源

1. 进入租房主页。
2. 点击右上角“+”或待办页的“新增房源”。
3. 填写小区名称、房号、租金。
4. 补充地址、户型、面积、业主信息。
5. 点击“提交审核”。
6. 查看新生成的房源详情。
7. 进入待办页面查看“新房源待审核”任务。

### 流程三：出租员处理出房

1. 进入“我的”页面。
2. 点击“切换身份”，选择“出租员”。
3. 进入“经营待办”。
4. 切换到“出房”或“合同”分类。
5. 点击房源查看详情。
6. 返回待办页面，点击“处理”完成任务。

### 流程四：投资人查看收益

1. 进入“我的”页面。
2. 切换身份为“投资人”。
3. 进入“统计”页面。
4. 查看收入、支出、利润、出租结构。
5. 回到房源列表查看不同房源的月租和收益率。

## 8. 常见问题

### 8.1 打开后是空白页怎么办？

建议按顺序处理：

1. 微信开发者工具点击“清缓存并编译”。
2. 确认当前编译入口是 `pages/index/index`。
3. 检查控制台是否有 WXML、JS 或网络报错。
4. 若要使用租房后端，确认 `http://localhost:3010/health` 可以访问。

### 8.2 原 AI 功能在哪里？

首页“应用中心”中保留了两个原功能入口：

- AI P 图 / 生成图片
- AI 宠物面相

它们继续使用线上 AI 后端：

```text
https://aipet.szlucky.online
```

### 8.3 租房后端端口为什么是 3010？

因为本机 `3000` 端口可能已经被原 AI 后端或其他服务占用。租房 MVP 使用 `3010`，避免互相覆盖。

## 9. 接口清单

租房 MVP 后端主要接口：

- `GET /health`
- `GET /api/bootstrap`
- `GET /api/dashboard`
- `GET /api/rooms`
- `GET /api/rooms/:id`
- `POST /api/rooms`
- `PATCH /api/rooms/:id`
- `GET /api/tasks`
- `PATCH /api/tasks/:id/complete`
- `GET /api/finance/summary`
- `POST /api/demo/reset`
